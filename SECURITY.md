# Security Policy

## Supported Versions

Security fixes are applied to the latest code on the default branch.

## Reporting a Vulnerability

Do not open public issues for security vulnerabilities.

Report privately to:
- theshovonsaha@gmail.com

Include:
- issue description
- reproduction steps or proof of concept
- potential impact
- mitigation ideas (if available)

Target response times:
- acknowledgement: within 3 business days
- status/remediation update: within 7 business days

## Responsible Disclosure

1. Validate and triage report.
2. Build and test fix.
3. Coordinate disclosure with reporter.
4. Publish advisory/changelog note as appropriate.
